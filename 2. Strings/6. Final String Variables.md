## âœ… **Final String Variables**
A `final` variable is replaced with its value at **compile time only if it is a compile-time constant**, meaning:

- It is **declared `final`**
- It is **assigned a value at declaration**
- The value is a **constant literal (not computed at runtime)**

ðŸ“Œ Such constants are **inlined by the compiler**.

```java
final String s = "Hello";
String result = s + "World";
```
| Memory Type | Objects Created | Explaination                              |
| ----------- | --------------- | ------------------------------------ |
| **SCP**     | **3**           | `"Hello"`, `"World"`, `"HelloWorld"` |
| **Heap**    | **0**           | No runtime objects created           |

```java
final String s = new String("Hello");
String result = s + "World";
```
| Memory Type | Objects Created | Explaination                                              |
| ----------- | --------------- | ----------------------------------------------------- |
| **Heap**    | **2**           | `"Hello"` (from new), `"HelloWorld"` (runtime concat) |
| **SCP**     | **2**           | `"Hello"`, `"World"`                                  |

- Since s is created using new, not a compile-time constant, so concatenation happens at runtime â†’ heap object created


```java
final String s;
s = new String("Hello");  // assigned later
String result = s + "World";
```
| Memory Type | Objects Created | Explaination                                             |
| ----------- | --------------- | ---------------------------------------------------- |
| **Heap**    | **2**           | `"Hello"` (via new), `"HelloWorld"` (runtime concat) |
| **SCP**     | **2**           | `"Hello"`, `"World"`                                 |
