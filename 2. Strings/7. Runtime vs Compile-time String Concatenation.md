### **ðŸŸ¢ Compile-time Concatenation**

Occurs **only when all operands are compile-time constants**.

âž¡ The result is computed at **compile time** and stored in the **String Constant Pool (SCP)**.

---

### **ðŸ”µ Runtime Concatenation**

Occurs when **at least one operand is *not* a compile-time constant** (even if it's `final` but not a literal).

Internally:

* A **`StringBuilder`** (or `StringBuffer`) is used
* **Intermediate strings are created on the heap**
* **Final concatenated string is created on the heap**
* **No string goes to SCP** unless `.intern()` is explicitly called

```java
String s = "geeks" + "for" + "geekies";
```

| Memory Type                    | Objects Created | Details                                                        |
| ------------------------------ | --------------- | -------------------------------------------------------------- |
| **Heap**                       | **0**           |  |
| **String Constant Pool (SCP)** | **4**           | `"geeks"`, `"for"`, `"geekies"`, `"geeksforgeekies"`           |

```java 
String s1 = "geeks";
String s2 = "for";
String s3 = "geekies";
String result = s1 + s2 + s3;
```
| Memory Type                    | Objects Created | Details                                                        |
| ------------------------------ | --------------- | -------------------------------------------------------------- |
| **Heap**                       | **1**           |  `"geeksforgeekies"` |
| **String Constant Pool (SCP)** | **3**           | `"geeks"`, `"for"`, `"geekies"`           |
